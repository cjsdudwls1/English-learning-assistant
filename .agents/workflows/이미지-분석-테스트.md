# 이미지 분석 파이프라인 로컬 테스트

이미지에서 문제를 추출하고 user_answer, correct_answer가 올바르게 인식되는지 검증하는 워크플로우.

## 사전 조건

- Node.js 22 이상 설치
- `cloud-functions/analyze-image/` 디렉토리에서 `npm install` 완료
- `.env.yaml`에 `GOOGLE_SERVICE_ACCOUNT_JSON` 설정 완료

## 실행 단계

### 1. 의존성 확인

```bash
cd cloud-functions/analyze-image
npm ls
```

의존성이 설치되지 않았다면:

// turbo
```bash
cd cloud-functions/analyze-image && npm install
```

### 2. 기본 테스트 실행 (기본 이미지 + 기대값 사용)

기본값:
- 이미지: `test_image/이어지는 지문/이어지는 지문/KakaoTalk_20251202_101043325_07.jpg`
- 기대 사용자답안: `4,5,2` (문제 25, 26, 27)
- 기대 실제답안: `5,3,5` (문제 25, 26, 27)

```bash
cd cloud-functions/analyze-image
node test-analyze.js
```

### 3. 커스텀 이미지 및 기대값으로 테스트

```bash
node test-analyze.js "<이미지 경로>" "<기대 사용자답안>" "<기대 실제답안>"
```

예시:
```bash
node test-analyze.js "../../test_image/이어지는 지문/이어지는 지문/KakaoTalk_20251202_101043325_07.jpg" "4,5,2" "5,3,5"
```

### 4. 상세 로그 모드

`--verbose` 플래그를 추가하면 Pass A/0/B의 전체 JSON 데이터를 출력:

```bash
node test-analyze.js --verbose
```

### 5. 결과 판독

테스트 결과는 다음과 같이 표시됨:

```
┌──────────┬──────────────┬──────────┐
│ 문제번호 │ 사용자답안   │ 실제답안 │
├──────────┼──────────────┼──────────┤
│ 25       │ 4            │ 5        │
│ 26       │ 5            │ 3        │
│ 27       │ 2            │ 5        │
└──────────┴──────────────┴──────────┘

  [PASS] Q25: user_answer(기대=4, 실제=4) OK | correct_answer(기대=5, 실제=5) OK
  [PASS] Q26: user_answer(기대=5, 실제=5) OK | correct_answer(기대=3, 실제=3) OK
  [PASS] Q27: user_answer(기대=2, 실제=2) OK | correct_answer(기대=5, 실제=5) OK

  ====> 전체 결과: PASS (모든 문제 일치)
```

- **PASS**: 기대값과 실제 AI 분석 결과가 일치
- **FAIL**: 불일치 항목 존재 → 프롬프트 또는 모델 로직 점검 필요

### 6. 종료 코드

| 코드 | 의미 |
|------|------|
| `0`  | 전체 PASS |
| `1`  | 하나 이상 FAIL |
| `2`  | 스크립트 오류 (인증 실패, 파일 없음 등) |

## 트러블슈팅

### 인증 오류
- `.env.yaml`의 `GOOGLE_SERVICE_ACCOUNT_JSON`이 유효한지 확인
- 서비스 계정에 Vertex AI 권한(`aiplatform.endpoints.predict`)이 있는지 확인

### 이미지 파일 없음
- 경로에 한글이 포함된 경우 따옴표로 감싸서 실행
- 상대경로는 `cloud-functions/analyze-image/` 기준

### bbox가 0개로 검출
- 이미지 해상도가 너무 낮거나 기울어진 경우 발생 가능
- `--verbose`로 Pass 0 원시 응답 확인

### user_answer가 null
- 필기가 너무 희미하거나 크롭 영역이 잘못 잡힌 경우
- `--verbose`로 Pass B 원시 응답 확인
