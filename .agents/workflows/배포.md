## 1. Supabase Edge Function 배포

### 프로젝트 정보

### 배포 명령어


- 작업 디렉토리는 프로젝트 루트(`English-learning-assistant/`)여야 함.

## 2. GCP Cloud Functions gen2 배포 (analyze-image)

### 프로젝트 정보
- **GCP 프로젝트 ID**: `gen-lang-client-0516945872`
- **리전**: `asia-northeast3`
- **런타임**: `nodejs22`
- **엔트리포인트**: `analyzeImage`
- **타임아웃**: `540초 (9분)`
- **메모리**: `1GiB`
- **소스 경로**: `cloud-functions/analyze-image/`
- **환경변수 파일**: `.env.yaml`
- **배포 URL**: `https://analyze-image-jg35qrg4wa-du.a.run.app`

### 배포 명령어

```bash
# cloud-functions/analyze-image 디렉토리에서 실행
cd cloud-functions/analyze-image

gcloud functions deploy analyze-image \
  --gen2 \
  --region=asia-northeast3 \
  --runtime=nodejs22 \
  --entry-point=analyzeImage \
  --trigger-http \
  --allow-unauthenticated \
  --timeout=540s \
  --memory=1GiB \
  --source=. \
  --env-vars-file=.env.yaml \
  --project=gen-lang-client-0516945872
```

// turbo

### 주의사항
- 반드시 `cloud-functions/analyze-image/` 디렉토리에서 실행해야 함. 프로젝트 루트에서 실행하면 소스 업로드 실패.
- `.env.yaml`에 `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `GOOGLE_SERVICE_ACCOUNT_JSON`이 포함되어 있어야 함.
- `--allow-unauthenticated`는 프론트엔드에서 직접 HTTP 호출하므로 필요.
- 배포 후 URL이 변경되지 않음 (기존 함수 업데이트).
