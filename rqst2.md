## 📋 커서AI 명령 스크립트 (순서대로 실행)

### **1단계: DB 스키마 확인 및 설계**
```markdown
supabase-mcp를 활용하여 현재 DB 스키마를 확인하고, 다음 작업을 수행하라:

1. 'taxonomy' 테이블의 모든 칼럼 확인 (특히 depth1~4, depth1~4_en, definition_ko/en, error_signals_ko/en, example_wrong/correct, related_rules, core_rule_ko/en)
2. 문제 풀이 시간 추적을 위한 새로운 테이블 또는 칼럼 설계 및 추가
   - 필요 필드: 문제ID, 사용자ID, 시작시간, 종료시간, 소요시간 등
3. 엣지 펑션 확인하여 언어 지원 관련 수정이 필요한 부분 파악
4. 전체 프로젝트 코드 검토하여 언어 전환 기능 구현 계획 수립
```

### **2단계: 언어 지원 기능 완성**
```markdown
언어 지원 기능(한국어/영어)을 완성하라:

1. 언어 전환 버튼('english' 버튼) 클릭 시 모든 UI/UX가 선택한 언어로 표시되도록 구현
2. 이미지 업로드 후 분석 기능: 선택된 언어에 따라 결과 출력
   - 한국어: taxonomy 테이블의 depth1~4 칼럼 참조
   - 영어: taxonomy 테이블의 depth1~4_en 칼럼 참조
3. 유사문제 생성 기능: 선택된 언어로 문제 생성
4. 유형 재분류 기능: 선택된 언어에 맞는 depth 칼럼 참조
5. 모든 버튼, 라벨, 메시지를 언어별로 표시
```

### **3단계: '?' 버튼 정보 표시 기능 추가**
```markdown
'?' 버튼 클릭 시 카테고리(depth) 정보를 보여주는 기능을 추가하라:

1. 사용자가 선택한 depth에 해당하는 taxonomy 테이블 데이터 조회
2. 선택된 언어에 따라 적절한 칼럼 표시:
   
   **한국어 선택 시:**
   - definition_ko
   - error_signals_ko
   - example_wrong
   - example_correct
   - related_rules
   - core_rule_ko
   
   **영어 선택 시:**
   - definition_en
   - error_signals_en
   - example_wrong
   - example_correct
   - related_rules
   - core_rule_en

3. 조회된 정보를 보기 좋게 포맷하여 한 화면에 모달 또는 팝업으로 표시
```

### **4단계: 예시 문장 생성 기능 추가**
```markdown
'예시 문장 생성' 기능을 추가하라:

1. class="flex gap-2" 근처에 '예시 문장 생성' 버튼 추가
2. 사용자가 카테고리(depth) 체크박스를 선택할 수 있도록 UI 구성
3. 버튼 클릭 시:
   - 선택된 depth에 해당하는 taxonomy 데이터 조회
   - LLM을 활용하여 해당 depth에 맞는 예시 문장 생성
   - 선택된 언어(한국어/영어)에 맞게 예시 문장 생성
4. 생성된 예시 문장을 사용자에게 표시
```

### **5단계: 유사문제 풀이 시간 추적 기능 구현**
```markdown
유사문제 생성 시 한 번에 하나씩만 풀 수 있도록 하고, 풀이 시간을 추적하는 기능을 구현하라:

1. 여러 개의 유사문제가 생성되어도 한 번에 하나의 문제만 표시
2. 문제 시작 시 타이머 시작
3. 문제 제출/완료 시 타이머 종료 및 소요 시간 계산
4. 1단계에서 설계한 DB 스키마에 따라 supabase에 풀이 시간 저장:
   - 문제 정보
   - 시작 시간
   - 종료 시간
   - 총 소요 시간
5. 다음 문제로 넘어가는 버튼 추가
```

## 💡 작업 순서 선정 이유

1. **DB 스키마 먼저**: 모든 기능이 DB 구조에 의존하므로 가장 먼저 확인 및 설계 필요
2. **언어 지원 두 번째**: 이후 모든 기능이 언어 설정에 영향을 받으므로 기반 구축 우선
3. **'?' 버튼 세 번째**: 언어 지원 완성 후 적용 가능
4. **예시 문장 네 번째**: 언어 지원과 taxonomy 조회 로직 완성 후 구현 가능
5. **시간 추적 마지막**: 가장 독립적인 기능이므로 다른 기능들이 안정화된 후 추가

이 순서대로 진행하면 의존성 문제 없이 효율적으로 작업할 수 있습니다!
