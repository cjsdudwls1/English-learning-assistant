1. 유사문제 생성 시 여러개의 문제가 생성되어도, 한번에 하나의 문제만 풀수 있도록 하고, 문제를 푸는데에 시간이 얼마나 걸리는지 체크해서 db에 저장. 이를 위해서 supabase-mcp를 활용하여 db스키마도 추가해야함

2. '예시 문장 생성' 기능 추가하라. 버튼 위치는 class="flex gap-2" 와 함께거나 가까운 위치로. 사용자는 예시문장을 보고 싶은 카테고리(문제유형 분류, depth) 의 체크박스를 선택 후 '예시 문장 생성' 버튼을 누를 것임. 그럼 해당되는 depth에 맞는 예시문장을 llm이 생성.

3. '?' 버튼 클릭시 4depth 별 정보 보여주기 기능 추가(칼럼명)

한글버전 시 나와야 할 정보에 참고해야할 'taxonomy' 테이블의 필드들은 다음과 같음:
definition_ko
error_signals_ko
example_wrong
example_correct
related_rules
core_rule_ko

영어버전 시 나와야 할 정보에 참고해야할 'taxonomy' 테이블의 필드들은 다음과 같음:
error_signals_en
example_wrong
example_correct
definition_en
label_en 
core_rule_en

사용자가 선택한 카테고리(depth)에 해당하는 supabase db에 저장되어있는 위 칼럼들을 받아 응용하여 사용자에게 해당 카테고리에 맞는 카테고리 정보 문장을 만들어서 사용자에 맞는 언어에 맞게 한 화면에 모두 보이도록 구현.

4. 언어 지원 기능(ko,en) 완성하라. 
현재는 웹에서 한/영 전환 버튼을 눌러도 아무 일도 일어나지 않음. 수정후에는 예를 들어 영어 버튼(ui 상 'english'버튼)을 누르면 모든 ui/ux는 영어로 표시되어야함. 
또한 사용자의 이미지 업로드 후 분석, 유사문제생성, 유형 재분류 기능의 결과물도 모두 영어로 output되어야 함. 특히 사용자의 이미지 업로드 후 분석, 유형재분류 기능은 supabase db에 저장 되어있는 'taxonomy' 테이블의 depth1~4를 참조하는데, 이 칼럼들의 값은 한글이기 때문에, 영어 값인 칼럼명 depth1~4_en 칼럼들을 참조해야함. 이외에도 애로사항 있을 수 있으니 작업 전 먼저 1. supabase-mcp로 db 확인 2. 프로젝트 코드 전체 확인 3. 엣지펑션확인 등의 작업을 거쳐 효과,효율적인 작업 계획을 세워서 작업하라.
