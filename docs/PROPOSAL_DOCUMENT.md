# AI 영어 문제 분석 시스템 - 기술 및 기능 정의서

> **프로젝트명**: AI 영어 문제 분석기 (English Learning Assistant)

---

## 1. 개발 개요

### 1.1 시스템 개요

본 시스템은 **사용자가 촬영한 영어 문제 이미지를 AI가 자동으로 인식, 분석, 채점**하는 웹 기반 학습 지원 솔루션임. 모바일 환경에 최적화되어 있으며, 학습자의 오답 패턴을 분석하여 맞춤형 학습 통계 및 유사 문제 생성 기능을 제공함.

### 1.2 핵심 기술 스택

| 구분 | 기술 | 선정 사유 |
|------|------|-----------|
| **프론트엔드** | React 19 + TypeScript | **안정성**: 대규모 커뮤니티와 장기 지원(LTS) 보장. **속도**: Virtual DOM 기반 고성능 렌더링. **확장성**: 컴포넌트 기반 아키텍처로 기능 확장 용이 |
| **빌드 도구** | Vite 5 | **속도**: ESBuild 기반 초고속 빌드(기존 대비 10~100배 향상). **안정성**: 프로덕션 환경 검증 완료. **확장성**: 플러그인 시스템 지원 |
| **스타일링** | Tailwind CSS 4 | **속도**: 빌드 타임 최적화로 CSS 번들 최소화. **확장성**: 유틸리티 클래스 기반 커스터마이징 용이 |
| **라우팅** | React Router 7 | **안정성**: 업계 표준 라우팅 라이브러리. **확장성**: 중첩 라우트, 지연 로딩 지원 |
| **차트/시각화** | Chart.js 4 + react-chartjs-2 | **안정성**: 10년 이상 검증된 차트 라이브러리. **확장성**: 다양한 플러그인 생태계 |
| **백엔드** | Supabase Edge Functions (Deno) | **속도**: 엣지 컴퓨팅으로 글로벌 저지연 응답. **안정성**: PostgreSQL 기반 트랜잭션 보장. **확장성**: 서버리스 아키텍처로 자동 스케일링 |
| **데이터베이스** | Supabase (PostgreSQL) | **안정성**: ACID 트랜잭션, Row Level Security(RLS) 지원. **확장성**: JSON/JSONB 타입으로 유연한 스키마 설계 |
| **AI 엔진** | Google Gemini API | **안정성**: Google Cloud 인프라 기반 99.9% 가용성. **속도**: 멀티모달 처리로 이미지+텍스트 동시 분석. **확장성**: 다중 모델 Failover 구조 적용 |
| **파일 저장소** | Supabase Storage | **안정성**: S3 호환 오브젝트 스토리지. **확장성**: CDN 연계로 글로벌 배포 지원 |
| **인증** | Supabase Auth | **안정성**: OAuth 2.0 / JWT 기반 보안 인증. **확장성**: 소셜 로그인 확장 가능 |
| **배포** | Netlify | **속도**: 글로벌 CDN을 통한 정적 파일 배포. **안정성**: 자동 HTTPS, 롤백 기능 |
| **PWA 지원** | vite-plugin-pwa | **확장성**: 네이티브 앱 수준의 사용자 경험 제공 |

### 1.3 시스템 아키텍처

본 시스템은 **4계층 구조**로 설계되어 있으며, 사용자 요청이 각 계층을 순차적으로 거쳐 처리됨. 아래에서 각 계층의 역할과 구성 요소를 상세히 설명함.

---

#### 계층 1: 사용자 접근 계층

사용자는 **모바일 또는 데스크톱 웹 브라우저**를 통해 시스템에 접근함. PWA(Progressive Web App) 기술을 적용하여 웹 브라우저에서도 네이티브 앱과 유사한 사용자 경험을 제공하며, 홈 화면에 앱 아이콘으로 설치할 수 있음.

---

#### 계층 2: 프론트엔드 계층 (Netlify CDN)

**기술 스택**: React 19 + TypeScript + Tailwind CSS + Chart.js

사용자 인터페이스를 담당하는 계층으로, Netlify CDN을 통해 전 세계에 빠르게 배포됨. 다음 4개의 주요 화면으로 구성됨:

| 페이지 | 주요 기능 |
|--------|----------|
| 메인 페이지 | 문제 이미지 업로드 및 분석 요청 |
| 통계 페이지 | 학습 현황 차트 및 표 시각화 |
| 세션 상세 페이지 | 분석된 문제 확인 및 정오답 라벨링 |
| 문제 생성/재도전 페이지 | AI 유사 문제 생성 및 풀이 |

**통신 방식**: 프론트엔드와 백엔드 간 통신은 REST API(HTTPS)를 사용하며, JWT(JSON Web Token) 기반 인증을 적용함.

---

#### 계층 3: 백엔드 계층 (Supabase Edge Functions)

**기술 스택**: Deno Runtime + Google Gemini API (다중 모델 Failover)

서버리스 Edge Function 방식으로 운영되며, 다음 6개의 핵심 기능 모듈로 구성됨:

| 함수명 | 기능 설명 |
|--------|----------|
| analyze-image | 업로드된 이미지를 AI로 분석하여 문제를 추출함 |
| generate-similar-problems | 기존 문제와 유사한 유형의 연습 문제를 AI가 생성함 |
| generate-example | 특정 문법/어휘에 대한 예문을 AI가 생성함 |
| generate-report | 학습 분석 리포트를 생성함 |
| reclassify-problems | 문제의 유형 분류를 재설정함 |
| _shared (공통 모듈) | AI 클라이언트, 에러 처리, 입력 검증 등 공통 기능을 제공함 |

---

#### 계층 4: 데이터 계층 (Supabase)

데이터 저장 및 관리를 담당하는 계층으로, 크게 데이터베이스와 파일 저장소 두 부분으로 구성됨.

**PostgreSQL Database**: 핵심 데이터를 관계형 모델로 저장함.

| 테이블명 | 저장 데이터 |
|----------|------------|
| sessions | 분석 세션 정보 |
| problems | 추출된 문제 데이터 |
| labels | 정오답 분류 라벨 |
| taxonomy | 문제 유형 분류 체계 |
| profiles | 사용자 프로필 정보 |
| generated_problems | AI가 생성한 문제 |
| problem_solving_sessions | 문제 풀이 기록 |

**보안**: Row Level Security(RLS)를 적용하여 사용자별 데이터가 완전히 격리됨.

**Supabase Storage**: 사용자가 업로드한 문제 이미지를 저장하는 오브젝트 스토리지임. `problem-images` 버킷을 사용하며, CDN을 통해 전 세계에 빠르게 이미지를 배포함.

---

#### 계층 5: AI 처리 계층 (Google Gemini API)

**다중 모델 Failover 구조**: 안정성을 확보하기 위해 5단계 백업 체계를 적용함.

| 우선순위 | 모델명 | 역할 |
|----------|--------|------|
| 1차 시도 | gemini-3-flash-preview | 기본 처리 모델 (가장 빠름) |
| 2차 시도 | gemini-2.5-flash | 1차 실패 시 대체 모델 |
| 3차 시도 | gemini-2.5-flash-lite | 2차 실패 시 경량 대체 모델 |
| 4차 시도 | gemini-2.5-pro | 3차 실패 시 고정밀 백업 모델 |
| 최종 백업 | gemma-3-27b-it | 모든 Gemini 모델 실패 시 최종 백업 (오픈소스 모델) |

**멀티모달 처리**: 하나의 API 호출로 이미지 OCR, 텍스트 분석, 문제 분류, 메타데이터 생성을 동시에 수행함.

### 1.4 아키텍처 설명

**3-Tier 아키텍처**로 설계되어 있으며, 각 계층은 명확하게 분리됨:

1. **프레젠테이션 계층 (프론트엔드)**: React 기반 SPA로 구현되어 Netlify CDN을 통해 전역 배포함. PWA 지원으로 모바일 홈 화면에 앱처럼 설치 가능함.

2. **비즈니스 로직 계층 (Edge Functions)**: Supabase Edge Functions를 활용하여 서버리스 방식으로 운영함. AI 분석, 문제 생성, 리포트 생성 등 핵심 연산을 처리하며, 다중 AI 모델 Failover 구조로 안정성을 확보함.

3. **데이터 계층 (Supabase)**: PostgreSQL 기반 관계형 데이터베이스로 문제, 세션, 라벨, 사용자 정보를 관리함. Row Level Security(RLS)를 통해 사용자별 데이터 격리를 보장함.

**비동기 백그라운드 처리**: 이미지 분석은 업로드 즉시 세션 ID를 반환하고, 실제 AI 분석은 백그라운드에서 진행됨. 이를 통해 사용자는 업로드 후 즉시 다른 작업을 수행할 수 있으며, 분석 완료 시 통계 페이지에서 결과를 확인할 수 있음.

## 2. 주요 기능 명세

### 2.1 핵심 기능 목록

| 기능명 | 기능 설명 | 사용자 가치 |
|--------|----------|-------------|
| **문제 이미지 업로드** | 최대 3장의 문제 이미지를 동시에 업로드하여 분석을 요청함 | 한 번에 여러 페이지의 문제를 분석하여 학습 효율을 높임 |
| **AI 자동 문제 인식** | 업로드된 이미지에서 문제 텍스트, 보기, 정답 등을 자동으로 추출함 | 수기로 문제를 입력할 필요 없이 촬영만으로 학습 기록이 생성됨 |
| **자동 채점** | 사용자가 표시한 정오답 마킹을 인식하여 자동으로 채점함 | 별도의 채점 작업 없이 즉시 결과를 확인할 수 있음 |
| **문제 유형 분류** | DB에 저장된 영어 문법/어휘 기준표(Taxonomy)에 따라 문제를 4단계 깊이로 분류함 | 어떤 유형의 문제에서 오답이 많은지 한눈에 파악할 수 있음 |
| **학습 통계 대시보드** | 정답률, 문제 유형별 분포, 기간별 추이를 차트로 시각화함 | 본인의 학습 패턴과 취약점을 객관적으로 분석할 수 있음 |
| **계층적 통계 분석** | 문제 유형을 4단계 깊이(대분류→중분류→소분류→세부유형)로 드릴다운하여 분석함 | 취약 영역을 세부적으로 파악하여 집중 학습이 가능함 |
| **유사 문제 생성** | 틀린 문제와 유사한 유형의 연습 문제를 AI가 자동으로 생성함 | 취약한 유형의 문제를 반복 학습하여 실력을 향상시킴 |
| **오답 문제 다시 보기** | 이전에 틀린 문제의 내용을 다시 확인할 수 있음 | 틀린 문제를 복습하여 동일한 실수를 방지함 |
| **문제 편집/수정** | AI가 인식한 문제 내용을 사용자가 직접 수정할 수 있음 | 인식 오류를 바로잡아 정확한 학습 기록을 유지함 |
| **문제 라벨링** | AI가 제안한 정오답 판정을 사용자가 검토하고 확정함 | 자동 채점의 오류를 보정하여 정확한 통계를 확보함 |
| **예문 생성** | 특정 문법/어휘 유형에 대한 이해를 돕는 예문을 AI가 생성함 | 개념 이해를 강화하여 응용력을 높임 |
| **문제 메타데이터 표시** | 난이도, 어휘 난이도, 문제 유형, 분석 정보를 상세히 제공함 | 문제의 특성을 파악하여 학습 전략을 수립함 |
| **다국어 지원** | 한국어, 영어 두 가지 언어로 인터페이스를 제공함 | 언어 선호에 따라 편리하게 사용할 수 있음 |
| **다크/라이트 모드** | 사용자 선호에 따라 화면 테마를 전환할 수 있음 | 눈의 피로를 줄이고 다양한 환경에서 사용할 수 있음 |
| **세션 관리** | 분석 세션의 상태(분석 중/완료/실패/라벨링 대기)를 확인하고 관리함 | 분석 진행 상황을 추적하고 필요 시 재시도할 수 있음 |
| **분석 실패 세션 관리** | 분석에 실패한 세션의 원인을 확인하고 정리(삭제)할 수 있음 | 실패 원인을 파악하여 다음 업로드 시 개선할 수 있음 |
| **이미지 회전** | 업로드 전 이미지의 방향을 회전하여 조정할 수 있음 | 촬영 방향이 잘못된 경우에도 올바르게 분석할 수 있음 |
| **문제 시험지 뷰** | 생성된 문제를 시험지 형태로 정리하여 출력할 수 있음 | 학습 자료로 활용하거나 오프라인에서 연습할 수 있음 |
| **사용자 프로필 관리** | 언어 설정, 계정 정보 등 사용자 설정을 관리함 | 개인화된 학습 환경을 구성할 수 있음 |

### 2.2 사용자 시나리오

#### 시나리오 1: 학생의 오답 노트 작성

1. **문제 촬영**: 학생이 학교에서 풀었던 영어 시험지를 스마트폰으로 촬영함
2. **이미지 업로드**: 메인 화면에서 "이미지 업로드" 버튼을 눌러 사진을 선택함 (최대 3장)
3. **분석 시작**: "AI 분석 시작하기" 버튼을 누르면 즉시 업로드 완료 메시지가 표시됨
4. **백그라운드 분석**: 앱을 닫아도 AI가 자동으로 문제를 인식하고 채점함
5. **결과 확인**: 통계 페이지에서 분석된 문제 목록과 정오답 현황을 확인함
6. **라벨 확정**: AI가 판단한 정오답을 검토하고, 필요 시 수정하여 확정함
7. **취약점 파악**: 통계 차트에서 오답률이 높은 문제 유형을 확인함
8. **유사 문제 연습**: 취약 유형에 대해 "유사 문제 생성" 기능으로 추가 연습함

#### 시나리오 2: 학부모의 자녀 학습 현황 파악

1. **접속**: 자녀 계정으로 로그인한 상태에서 통계 페이지에 접속함
2. **기간 설정**: 날짜 필터를 통해 특정 기간(예: 이번 달)의 학습 기록을 조회함
3. **전체 현황 확인**: 도넛 차트로 전체 정답/오답 비율을 확인함
4. **유형별 분석**: 막대 그래프를 클릭하여 취약한 문제 유형으로 드릴다운함
5. **세부 통계**: 4단계 깊이까지 분석하여 구체적인 취약 영역(예: 관계대명사 - 목적격)을 파악함
6. **학습 조언**: 파악된 취약점을 바탕으로 자녀에게 집중 학습 영역을 안내함

#### 시나리오 3: 교사의 학습 자료 제작

1. **문제 선택**: 통계 페이지에서 학생들이 자주 틀리는 유형을 확인함
2. **유사 문제 생성**: 해당 유형에 대해 AI 유사 문제 생성 기능을 실행함
3. **문제 검토**: 생성된 문제를 검토하고 필요 시 수정함
4. **시험지 출력**: 시험지 뷰 기능을 통해 정리된 형태로 출력함
5. **수업 활용**: 제작된 자료를 수업 시간에 활용함

#### 시나리오 4: 오답 문제 복습

1. **오답 필터**: 통계 페이지에서 "오답만 보기" 필터를 적용함
2. **문제 선택**: 복습하고 싶은 문제를 선택함
3. **문제 확인**: 이전에 틀렸던 문제의 내용과 보기를 다시 확인함
4. **유사 문제 생성**: 해당 유형의 AI 생성 문제로 추가 연습함

#### 시나리오 5: 분석 실패 처리

1. **실패 확인**: 분석 중 오류가 발생하여 "분석 실패" 상태가 표시됨
2. **오류 원인 확인**: 실패 카드에서 원인(예: 이미지 품질 불량, AI 모델 과부하 등)을 확인함
3. **세션 정리**: 해당 실패 세션을 삭제하여 목록을 정리함
4. **재업로드**: 더 선명한 이미지로 다시 촬영하여 새로 업로드함

#### 시나리오 6: 문제 편집 및 수정

1. **인식 오류 발견**: AI가 인식한 문제 내용 중 오류를 발견함
2. **편집 모드 진입**: 해당 문제의 "편집" 버튼을 클릭함
3. **내용 수정**: 문제 텍스트, 보기, 정답 등을 올바르게 수정함
4. **분류 수정**: 필요 시 문제 유형 분류도 수정함
5. **저장**: 변경 사항을 저장하여 정확한 학습 기록을 유지함

## 3. 확장성

본 시스템은 다음과 같은 기능까지 확장 가능한 구조로 설계함:

- **다과목 지원**: 현재 영어 문제에 특화되어 있으나, 문제 유형 분류 체계를 확장하여 수학, 과학 등 다른 과목의 문제 분석 기능까지 확장 가능함
  
- **학급/학교 단위 통계**: 현재 개인 사용자 단위의 통계를 제공하나, 그룹 기능을 추가하여 학급 전체의 학습 현황을 분석하는 기능까지 확장 가능함

- **학습 경로 추천**: 축적된 오답 데이터를 기반으로 개인 맞춤형 학습 경로를 AI가 추천하는 기능까지 확장 가능함

- **실시간 협업 학습**: 실시간 통신 기능을 추가하여 교사-학생 간 실시간 문제 풀이 세션 기능까지 확장 가능함

- **음성 인식 입력**: 음성 답변 인식 기능을 추가하여 말하기 연습 및 답변 입력 기능까지 확장 가능함
