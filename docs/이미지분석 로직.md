현재 파이프라인에서 학생이 필기한 동그라미(O), 엑스(X), 브이(V) 표시 같은 **`user_answer`(사용자 답안)는 오직 "Gemini"가 잡고 있습니다.**

정확히 단계별로 어떤 AI가 무얼 하는지 설명해 드릴게요.

### 🔍 현재 파이프라인의 명확한 역할 분담

**1단계 (Pass A - 텍스트 및 구조 추출)**
* **Document AI:** 이미지의 **"인쇄된 텍스트(문제, 지문, 선택지 글자)"**만 아주 또렷하고 정확하게 읽어냅니다.
* **Gemini:** Document AI가 읽어온 텍스트를 받아서 "아, 이게 28번 문제구나, 여기서부터 여기까지가 지문이네" 하고 JSON 구조(뼈대)로 조립합니다.

**2단계 (Pass 0 - 바운딩 박스 식별)**
* **Gemini:** 전체 시험지 이미지를 보고 "선택지 1, 2, 3, 4, 5번이 모여있는 영역(정답 마킹 영역)이 대충 x좌표 몇, y좌표 몇이구나"하고 찾아냅니다. 
* (우리 코드가 이 좌표를 바탕으로 정답 영역만 작게 싹둑 자릅니다. = 크롭 이미지)

**3단계 (Pass B - 필기 인식 및 최종 정답 도출) ← `user_answer` 추출 단계**
* **Gemini:** 싹둑 잘린 "조그마한 크롭 이미지"를 뚫어져라 쳐다봅니다.
* 이 단계에서는 Document AI는 아예 관여하지 않습니다. 오직 Gemini의 **시각적 추론 능력(Vision)**에만 의존하여 "어? 3번 위에 연필로 슥슥 그어진 자국이 있네. 학생 답은 3번이구나" 하고 찾아냅니다.

### 💡 왜 _09.jpg 같은 흐린 이미지에서 필기를 못 잡는지 이해가 되실 겁니다.

`user_answer`를 잡는 주체가 **Gemini**이다 보니 발생하는 현상입니다.
* Gemini는 원래 텍스트를 읽고 맥락을 파악하는 데 특화된 언어 모델입니다.
* 이미지를 보는 눈(Vision)이 있긴 하지만, 전문적인 '도형/손글씨 판독기'가 아니기 때문에, _09.jpg처럼 화질이 흐리거나 연필로 대충 뭉개놓은 표시는 "이게 얼룩인지 표기인지" 확신하지 못하고 놓쳐버립니다(null 반환).

### 결론
* **인쇄된 한글/영어 텍스트** = Document AI + Gemini 합작
* **학생이 손으로 표시한 채점/마킹(user_answer)** = 순수 **Gemini** 단독 시각 추론

만약 앞서 말씀하신 **"Document AI 파인튜닝"**을 하게 된다면, 이 `user_answer`를 잡는 역할까지 **Gemini에게서 뺏어서 Document AI에게 넘겨주는 셈**이 됩니다. 전문 OCR 엔진이 손글씨와 채점 기호까지 전문적으로 학습하여 찾아내게 되니까 필기 인식의 정확도가 획기적으로 올라가게 되는 원리입니다.
